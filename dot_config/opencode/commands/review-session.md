---

请对我们这次完整的对话与任务执行过程进行结构化复盘，并将结论固化到 `AGENTS.md` 中。

**复盘要求：**
1. 识别以下四类不流畅场景（若有）：
   - 指令误解：需要我打断或纠正你的理解。
   - 上下文丢失：你忘记了之前的关键信息或约束。
   - 流程冗余/偏离：执行步骤冗余、跳跃或不符合预期，需要我介入调整。
   - 架构违规：代码实现违反了项目的架构约定（如目录职责、模块边界等）。
2. 对每个实际出现的场景，按以下格式输出：
   - **问题**：具体发生了什么？
   - **原因**：根本原因分析（如指令模糊、上下文窗口限制、缺少前置规划、架构未定义等）。
   - **改进方案**：可执行的改进措施，规则应具体、无歧义，例如：“在执行任何文件写入前，必须先输出计划并等待用户确认”，而不是“要更小心”。

**写入规则：**
- 定位到当前目录的 `AGENTS.md` 文件。
- 找到或创建 `# 复盘沉淀` 一级标题。
- 在该标题下，以 `## YYYY-MM-DD`（今天的日期）为二级标题，追加本次复盘内容。
- 若已有相同日期的子标题，则合并更新；若改进规则与历史条目重复，则合并或标注“已更新”。
- 保留原有所有内容，不得删除或修改非本次复盘的内容。

**输出格式示例：**
```markdown
# 复盘沉淀
## 2026-01-15
### 指令误解
  问题：你要求我列出所有修改过的文件，我却直接开始总结内容。
  原因：指令中的“列出”被我忽略，只记住了“总结”。
  改进：当用户使用‘列出’、‘仅展示’等词时，禁止执行任何修改操作，只输出列表。
### 上下文丢失
  问题：对话中期我忘记了项目使用 Python 3.10，生成了 f-string 调试语句导致语法错误。
  原因：上下文窗口滚动后丢失了初始约束。
  改进：项目必须兼容 Python 3.10，禁止使用 3.11+ 特性（如自省略号等）。
```
