---

请对我们这次完整的对话与任务执行过程进行结构化复盘，并将结论固化到 `AGENTS.md` 中。

**复盘要求：**
1. 识别以下四类不流畅场景（若有）：
   - 指令误解：需要我打断或纠正你的理解。
   - 上下文丢失：你忘记了之前的关键信息或约束。
   - 流程冗余/偏离：执行步骤冗余、跳跃或不符合预期，需要我介入调整。
   - 架构违规：代码实现违反了项目的架构约定（如目录职责、模块边界等）。
2. 对每个实际出现的场景，先在脑中完成复盘（可包含问题/原因/权衡），但**最终只输出可执行的规则**：
   - 规则必须是**可操作、无歧义、可检验**的句子（能直接约束后续行为）。
   - 规则尽量使用"必须/禁止/只要…就…"等强约束措辞。
   - **禁止**在输出中展开"问题、原因、改进方案"的三段论细节（只保留规则）。

**写入规则：**
- 定位到当前目录的 `AGENTS.md` 文件。
- 找到或创建 `# 项目规则` 章节。
- 将本次沉淀的规则**合并追加**到该章节下（去重后再写入）。
- 保留原有所有内容，不得删除或修改非本次复盘相关的内容。

**输出格式示例：**
```markdown
# 项目规则
- 当用户使用"列出/仅展示/只输出"时：禁止执行任何修改操作，只输出所需列表。
- 项目必须兼容 Python 3.10：禁止使用 3.11+ 特性。
```
