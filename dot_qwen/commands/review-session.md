description: 分析本次会话中的配合不畅场景，总结原因并写入 AGENTS.md 或 README.md 以避免重复问题。
---

请对本次对话进行结构化复盘，将结论固化到项目文件中。

**复盘要求：**
1. 识别以下四类不流畅场景（若有）：
   - 指令误解：需要我打断或纠正你的理解。
   - 上下文丢失：你忘记了之前的关键信息或约束。
   - 流程冗余/偏离：执行步骤冗余、跳跃或不符合预期，需要我介入调整。
   - 架构违规：代码实现违反了项目的架构约定。
2. 对每个场景判断产出类型：
   - **项目事实**（AI 不知道就会犯错的事实信息）→ 写入 `README.md`
   - **协作规则**（约束 AI 行为的可操作规则）→ 写入 `AGENTS.md`
3. 同时检查 `AGENTS.md` 中是否已有规则："处理本项目任务前，必须先读 README.md"，若无则补充。

**过滤标准（禁止写入）：**
- 常识性规则（如"ruff 只检查 Python 文件"）
- 仅本次会话有效的规则
- 已被项目事实覆盖的内容
- 描述具体 bug 的规则（应修复代码，不写规则绕过）

**写入规则：**
- **协作规则** → AGENTS.md 的 `# 项目规则` 章节，合并追加并去重。
- **项目事实** → README.md 适当章节（创建或追加到相关段落），保持文档结构清晰。
- 保留原有所有内容，不得删除或修改非本次复盘相关的内容。

**产出格式：**
规则使用"必须/禁止/只要…就…"等强约束措辞，无歧义、可检验。
禁止展开"问题、原因、改进方案"三段论细节。
